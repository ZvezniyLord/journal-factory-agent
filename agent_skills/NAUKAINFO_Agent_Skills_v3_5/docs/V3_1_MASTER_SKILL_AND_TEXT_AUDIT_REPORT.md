# Поглиблений аудит текстової цілісності журналу 136 v32

## Метод

- 24 статті розділено за реальними межами `pageBreakBefore` і стилем назви.
- 22 статті порівняно безпосередньо з вихідними DOCX.
- 2 старі `.doc`-статті перевірено через source PDF як fallback.
- Порівняння виконано після назви статті, з урахуванням тексту таблиць і text box у OOXML.
- Стандартні службові зміни винесено в whitelist: маркери references, `URL:`/`DOI:`, нумерація references, очищення шапки.

## Результат

- Порівняно статей: **24**.
- Підтверджених непогоджених видалень або замін авторського тексту: **0**.
- Статей зі статусом `BLOCKED`: **0**.
- Статей зі статусом `REVIEW`: **1**.

Для 22 нативних DOCX не виявлено підтвердженої втрати звичайного редагованого тексту тіла. Раніше PDF-only перевірка давала багато хибних спрацювань через переноси, дефіси, URL і текст усередині графічних об’єктів.

Є один залишковий випадок `REVIEW`: стаття Novak Natalya. Підписи `Abb. 1...` та `Abb. 4...` у source PDF і фінальному DOCX потрапляють у різний машинний порядок через legacy drawing/anchor, хоча обидва підписи візуально присутні у фінальному рендері (сторінки 66 і 69). Це не підтверджена втрата, але й не може бути автоматично оголошено повним XML-текстовим PASS без object-level доказу.

## Висновок

**Підтвердженої втрати звичайного редагованого авторського тексту не знайдено.**

Водночас абсолютне твердження «цілісність доведена на 100%» поки некоректне: текст, який існує лише всередині legacy OLE/shape/raster objects, потребує окремого per-article media/hash/order/render gate. Тому загальний статус поглибленого текстового аудиту: **REVIEW**, а не безумовний `PASS`.
